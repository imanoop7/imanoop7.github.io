E-Books Structure Guide#

This guide explains how to use the e-books section of your Hugo blog.

Directory Structure#

content/e-books/
├── _index.md                    # Main e-books landing page
└── your-book-name/              # Individual book folder
    ├── _index.md                # Book overview page
    ├── chapter-1-title.md       # Chapter 1
    ├── chapter-2-title.md       # Chapter 2
    └── ...                      # Additional chapters

static/downloads/                # PDF files storage
├── your-book-chapter-1.pdf
├── your-book-chapter-2.pdf
└── ...

Adding a New Book#

  1. Create a new folder under content/e-books/ with your book name (use lowercase and hyphens)
  2. Add an _index.md file in the book folder with this frontmatter:
---
title: "Your Book Title"
layout: "book"
url: "/e-books/your-book-name/"
summary: "Brief description of your book"
description: "Longer description for SEO"
author: "Your Name"
publishDate: "2024-01-01"
tags: ["tag1", "tag2", "tag3"]
---

Adding Chapters#

Create chapter files with this frontmatter structure:

---
title: "Chapter Title"
date: 2024-01-01
layout: "chapter"
type: "chapter"
chapter: 1                       # Chapter number for ordering
summary: "Chapter description"
tags: ["tag1", "tag2"]
toc: true                        # Enable table of contents
readingTime: 10                  # Estimated reading time
downloadLink: "/downloads/chapter.pdf"  # Optional PDF download
---

PDF Integration#

  1. Upload PDF files to the static/downloads/ directory
  2. Reference them in chapter frontmatter: downloadLink: "/downloads/filename.pdf"
  3. The download button will automatically appear on the chapter page

Features#

  • Automatic Navigation: Chapters are automatically grouped by book
  • Chapter Ordering: Use the chapter parameter to control order
  • PDF Downloads: Optional PDF versions of chapters
  • Responsive Design: Works on all devices
  • Search Integration: Chapters are included in site search
  • SEO Friendly: Proper meta tags and structured data

Example#

See the sample-book folder for a complete example of how to structure your content.

Tips#

  • Use consistent naming conventions for files and folders
  • Keep chapter numbers sequential (1, 2, 3, etc.)
  • Add meaningful tags for better categorization
  • Include summaries for better user experience
  • Test PDF links to ensure they work correctly
